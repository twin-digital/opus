Part 4

Layout primitives

Containers — cards, tabs, tables, modals — that hold content. Specified as configurable shells, composed many times downstream.

Cards

A card is a bordered, rounded container. Compose, don't proliferate variants — the FE team should build the card they need from these rules, not request a new specimen for each surface.

Anatomy

Eyebrow

Title

Body content. Use whatever shape fits — prose, table, chips, key/value list.

Optional regions

  • Eyebrow — text-xs uppercase tracking-wider text-zinc-500. Use for label categories ("Alerts", "Triages last 24h").
  • Title — text-base semibold. Skip when the body content (a big number, a chart) is itself the headline.
  • Action slot — top-right of the header; menu (more-horizontal) or a single button.
  • Body — p-4 default, p-3 dense, p-5 spacious.
  • Footer — only when the card carries an action; tinted bg (bg-zinc-50 dark:bg-zinc-900/50) + top border.

Variants

Default

Neutral container. Use for everything that isn't semantically loaded.

Alert

Tinted border + bg. Pick the status color that matches (red / amber / emerald).

Muted

No border, tinted fill. For grouping content inside a larger card or panel.

Composition rules

Cards stack with gap-4 in columns or grid grid-cols-N gap-4 in grids. Match column count to viewport; collapse to one column below md.

For number-dominated cards (Dashboard tiles), use a small uppercase eyebrow and let the value (text-3xl semibold) be the headline. No title.

For action cards (forms, configuration panels), use the full header + body + footer pattern with the action footer tinted.

For alert cards, render only when non-zero. An always-on red card normalizes alarm and stops being scannable.

Tabs

Underline style. Active tab uses the violet accent; counts render in a muted pill on the right of the label.

Tables

TanStack Table. Column headers double as sort buttons. Comfortable density (~48px row, 15–20 rows above the fold at 1080p). Pagination lives in the footer.

Column-header sort affordance

Part 6

Inbox & Messages

The centerpiece. Inbox is the cross-Account browse surface; Message detail is "why did Grinbox do that?" with three tabs (Overview, Tags, Triage history).

Inbox header

Search input + applied filter chips + available filter chips, on a single row above the table.

Account: Personal

Account badge & picker

Per-account display badge — a glyph (default mail) in a Tag-palette color — marks a Message's source account in the Inbox hanging indent and the account list. Set per account in settings via an icon grid + a color row (with a neutral option); colors reuse the Tag-chip hues. Selected option carries a ring.

Badge — icon / color combos

Icon picker (briefcase selected)

Color picker (sky selected · briefcase glyph)

Inbox row

A per-Message account icon sits in a hanging indent on the far left — top-aligned so it spans the metadata and the tag row, marking the source account at a glance when multiple accounts interleave in one list. It defaults to a mail glyph; the icon + color are set per-account in account settings. To its right: the primary row (From · Subject · Status · Time; default sort Time/received desc), then Tags on their own full-width wrapping row with a bit of breathing room above. Messages with no Tags show only the primary row. The two accounts below (neutral vs. violet) show how interleaved rows stay scannable.
